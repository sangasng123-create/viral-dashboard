후임자 가이드 — 개발 몰라도 됩니다
==================================

이 문서는 개발을 전혀 모르는 사람이 이 대시보드를 넘겨받아
운영할 수 있도록 쉬운 말로 쓴 안내서입니다.
(더 자세한 건 HANDOVER.md, 매칭 규칙은 MATCHING.md 참고)


먼저 — 일은 딱 두 종류뿐입니다
------------------------------

  종류 1) 평소 일 = 구글시트에 데이터 입력
          → 대시보드가 알아서 갱신됨. 코드도 깃도 안 건드림. (99%가 이것)

  종류 2) 가끔 일 = 대시보드 화면/기능 자체를 바꾸고 싶을 때
          → 이때만 코드 수정 + "깃푸시"가 필요함. 혼자 끙끙대지 말고
            개발 도움(개발자 또는 Claude 같은 AI 도구)을 받으면 됨.

즉, 평소에는 구글시트만 만지면 됩니다. 겁먹을 것 없습니다.


처음 한 번만 하는 셋업
----------------------

  (1) 깃허브 계정 만들기
      - github.com 접속 → Sign up → 계정 생성 (무료)
      - 만든 아이디(username)를 전임자에게 알려주기 → 저장소를 넘겨받기 위함

  (2) 저장소 넘겨받기
      - 전임자가 "이전"을 보내면 이메일이 옴 → 메일에서 Accept(수락) 클릭
      - 이제 이 코드 저장소가 내 것이 됨

  (3) 대시보드 배포하기 (인터넷에 띄우기)
      - share.streamlit.io 접속 → 위에서 만든 깃허브 계정으로 로그인
      - "New app" → 내 저장소(viral-dashboard) 선택 → 파일은 app.py → Deploy
      - 잠시 후 "비밀번호를 설정하세요" 안내가 뜨면:
          앱 관리 → Settings → Secrets → 아래 한 줄 붙여넣고 Save

              app_password = "atwo1926"

      - 이건 한 번만 하면 계속 유지됨

  (4) 공용 구글계정 로그인 받기
      - 데이터(완료시트·성과DB)를 입력하려면 회사 공용 구글계정이 필요
      - 회사에서 로그인 정보를 받기


평소 운영 — 이것만 알면 됩니다
------------------------------

  - 구글 스프레드시트에 데이터를 입력/수정한다.
  - 대시보드는 5분마다 자동으로 시트를 다시 읽는다.
    (사이드바의 "원본 다시 불러오기" 버튼을 누르면 즉시 갱신)
  - 코드나 깃은 전혀 건드릴 필요 없다.

  ※ 데이터를 잘 적는 규칙(NT 파라미터, 닉네임 일치 등)은 MATCHING.md 참고.
    이게 대시보드 정확도의 핵심입니다.


"깃푸시"가 뭔가요? (코드를 바꿀 때만 나오는 말)
----------------------------------------------

쉽게 비유하면:

  - 깃허브 = 코드의 "원본 보관함" + "자동 배포 스위치"
  - 코드 파일을 고친 뒤 "깃푸시"를 한다는 것은
    = "고친 내용을 보관함에 저장하고, 배포 스위치를 누르는 것"
  - 푸시하면 Streamlit이 자동으로 새 코드로 대시보드를 다시 띄운다.

즉 푸시 = "저장 + 반영" 버튼이라고 생각하면 됩니다.

하지만 이건 코드를 직접 바꿀 때만 필요하고, 개발 지식이 있어야 합니다.
화면이나 기능을 바꾸고 싶으면:
  - 개발자에게 요청하거나
  - Claude Code 같은 AI 도구에게 "이렇게 바꿔줘"라고 부탁하면
    코드 수정과 깃푸시까지 대신 해줍니다.

혼자 코드를 만지려고 애쓸 필요 없습니다.
